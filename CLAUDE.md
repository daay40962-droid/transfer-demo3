# CLAUDE.md — проект «Трансфер Север»

## Что это
Демо-лендинг пассажирских перевозок по маршруту Красноярск — Северо-Енисейский.
Цель версии — показать заказчику (ИП Ежкова Е.М.), что попали в тематику и стиль.
Не финальный продукт: реквизиты реальные, но MAX-бот и тексты политик — заглушки.

## Стек
- Чистый HTML + CSS + JavaScript. Без фреймворков и сборщиков.
- Приём онлайн-заявки — через Netlify Function (`/api/send-telegram` → Telegram Bot API).
- Принцип mobile-first. Главная цель — телефон.
- Хостинг демо с рабочей онлайн-заявкой — Netlify. GitHub Pages / обычный static-hosting не запустит функцию.

## Порядок работ
Сайт собираем **по частям, не весь сразу**. Этапы и их статусы — в `ROADMAP.md`.
После каждого этапа — показ Коляну и согласование, потом следующий этап.
Начинать с актуального незакрытого этапа в ROADMAP.

## Где данные
Все редактируемые данные (контакты, точки маршрута, цены) — в объекте `siteData`
вверху `index.html`. Менять там, не трогая вёрстку. Эталон структуры — раздел 19 в `tz-FINAL.md`.
Публичную ссылку/username Telegram диспетчера менять там же: `siteData.contacts.telegram`.
Токен Telegram-бота и chat_id в `index.html` не писать никогда — только в переменных окружения Netlify.

## Запуск сайта

Сборки нет — чистый HTML/CSS/JS, открывается напрямую.

**Вариант 1 — Python (встроен в систему, не требует Node):**
```bash
python -m http.server 5500
```
Сайт: http://localhost:5500

**Вариант 2 — live-server (авто-перезагрузка при изменениях, нужен Node):**
```bash
npx -y live-server --port=5501 --no-browser
```
Сайт: http://localhost:5501

Важно: `python -m http.server` и `live-server` показывают сайт, но не запускают `/api/send-telegram`.
Для проверки отправки заявки локально нужен Netlify dev:
```bash
npx -y netlify-cli dev
```

Конфиг для `/run` скилла — `.claude/launch.json`.

## Перед деплоем
- В `siteData.contacts.telegram` указать реальный username или ссылку диспетчера Telegram.
- В Netlify → Site settings → Environment variables задать `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.
- Проверить, что секреты не попали в `index.html`, `netlify.toml`, тесты или документацию.
- Прогнать проверки:
```bash
node --test tests\send-telegram.test.js tests\index-static.test.js
```
- Проверить отправку онлайн-заявки через Netlify dev или preview deploy; на обычном `localhost:5501` функция не работает.
- Перед отправкой заказчику проверить `policy.html`: тексты политик всё ещё заглушки и требуют юридической проверки.

## Правила работы
